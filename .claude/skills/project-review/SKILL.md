# Project Review Skill

Проводи техническую ревизию проекта для выявления проблем качества и технического долга.

## Что проверять

### 1. Соответствие архитектуре
- Прочитай `00_docs/architecture/overview.md`
- Прочитай все ADR в `00_docs/architecture/decision_*.md`
- Проверь соответствие кода архитектурным решениям
- Выяви нарушения принятых паттернов

### 2. Технический долг
- Дублирование кода
- Dead code (неиспользуемые функции, импорты)
- Временные решения (TODO, FIXME, HACK)
- Отсутствие тестов
- Устаревшие зависимости

### 3. Качество кода
- Прочитай `00_docs/standards.md`
- Проверь соблюдение стандартов
- Сложные функции (>50 строк)
- Отсутствие документации
- Неочевидные решения без комментариев

### 4. Структура проекта
- Логичность организации файлов
- Консистентность именования
- Правильность размещения по папкам

## Процесс ревизии

1. **Определи область:**
   - Весь проект или конкретные модули
   - Фокус на архитектуре, качестве или долге

2. **Проведи проверку:**
   - Используй Read для чтения файлов
   - Используй Grep для поиска паттернов
   - Используй Glob для обзора структуры

3. **Зафиксируй результаты:**
   - Добавь секцию в конец релевантного файла:

```markdown
## Revision <N>

**Дата:** YYYY-MM-DD

**Область:** [Что проверялось]

**Проверено:**
- [Область 1]
- [Область 2]

**Выявлено:**
- [Проблема 1 с примерами]
- [Проблема 2 с примерами]

**Рекомендации:**
- [Действие 1]
- [Действие 2]

**Приоритет:**
- Критично: [проблемы требующие немедленного внимания]
- Важно: [проблемы для ближайших итераций]
- Желательно: [улучшения для будущих итераций]
```

4. **Сформируй задачи:**
   - Для критичных проблем предложи добавить задачи в `backlog.md`
   - Укажи приоритет и примерный объем работ

## Примеры проблем

**Нарушение архитектуры:**
```
В 02_src/module_a/handler.py напрямую обращаемся к БД, 
хотя по ADR-003 должны использовать repository layer.
```

**Технический долг:**
```
В 02_src/utils/helper.py функция process_data() имеет 150 строк
с вложенными циклами и дублирует логику из module_b.
```

**Нарушение стандартов:**
```
В 02_src/module_c/ используется camelCase, хотя по standards.md
требуется snake_case для Python.
```
